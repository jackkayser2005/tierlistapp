   101→- Added 4 new socket handlers inside `io.on('connection')`, all wrapped in try/catch with `[event]`-prefixed logging:
   102→  - `vote:start { roomId, itemId, item }` — validates roomId/itemId are non-empty strings and item is a plain object (rejects null/arrays); requires `room.host === socket.id` else `room:error`; sets `room.vote`, broadcasts `vote:state` (active) to the whole room including sender.

   105→  - `vote:cancel { roomId }` — host-only; clears `room.vote`, broadcasts inactive `vote:state`.
   106→- Updated the existing `disconnect` handler: after `forgetRoom` + `presence:update` per affected room, if `room.vote` is active and `room.vote.votes.has(socket.id)`, deletes the voter's entry and broadcasts a fresh `vote:state` so the tally updates live.
